Los datos - Pendigit¶

He decidido trabajar con los datos de Pendigit, obtenidos del repositorio de bases de datos de UCI. En el sitio web de los datos, se explica que fueron obtenidos de 44 diferentes personas, a las cuales se pidi que escribieran de manera aleatoria 250 nmeros. Esto se realiz en un dispositivo tipo tablet que mide las coordenadas en las cuales se encuentra la pluma, cada 100 milisegundos. Se eliminaron los primeros intentos, debido a que se consider que los escritores no estaban familiarizados con la tecnologa.

Casi cualquier dispositivo inteligente cuenta con una pantalla tctil, as que es completamente factible y aplicable al mundo real - en lo personal, yo tomo mis apuntes a mano en una tablet pues es sumamente complicado hacer grficas y frmulas en ella. Sin embargo, me encantara que tales apuntes fueran "buscables", poder buscar en qu parte hablo de un cierto tema. Por otro lado, como ciertos dispositivos suelen tener un agente humano que puede corregir a la mquina si se equivoca, adems de que no cuentan con tanta RAM, es preferible obtener rapidez en el modelo de clasificacin.

Los datos son presentados en ocho pares de coordenadas (16 variables numricas) que pertenecen al rango [0,100] y una categrica que pertenece al rango [0,1,...,9]. Podemos echar un primer vistazo a los datos:

#probar con diferentes C para ver la que d menor error
c_values=(0.01,0.5,1,5,10,100,1000,100000)
trues_svm=[]
for i in range(len(c_values)):
    clf_1=SVC(C=c_values[i],kernel='rbf')
    clf_1.fit(train_x,train_y)
    prediction = clf_1.predict(test_x[0:len(test_x)])
    trues_svm.append(sum(test_y==prediction))
print("Aciertos con SVM, para cada valor de C propuesto: " + str(trues_svm))

Aciertos con SVM, para cada valor de C propuesto: [365, 365, 479, 498, 498, 498, 498, 498]

Como podemos observar, parece el nmero de aciertos converge a un mximo al aumentar C - esto significa, generalmente, overfitting. A pesar de que el modelo se est esforzando por clasificar correctamente cada punto, presenta un 14.23% de precisin al predecir nuevos valores - esto es porque, en general, las SVM separan dos clases. Tomando en cuenta que una clase es el 10% de los datos, el desempeo de este algoritmo es, a lo ms, complicado de interpretar; adems de que se necesitaran 10 SVM para separar correctamente cada una de las clases, y esto es muy complejo.

Conclusiones¶

Comenc este proyecto sabiendo que era una base de datos fcil, y que era precisamente el tipo de problema que podra ser resuelto de una manera espectacular por K-vecinos cercanos - tiene todo el sentido del mundo que todos los doses se parezcan, y que est bastante lejos de los ochos.

Una cosa curiosa que encontr mientras jugaba, fue que SVM se porta terriblemente mal con todos los datos, alcanzando un accuracy de 14%, pero si usamos solamente el primer par de datos, este accuracy sube a 43%. Claro, este nmero no est siquiera cerca de poder competir con los otros dos modelos, pero es una mejora enorme. Probablemente est encontrando dos clases de nmeros, los que se empiezan ms bien a la derecha (2,3,4,7), o al centro (0,1,8) o a la izquierda (5,6,9).

Creo que lo ms importante que podemos obtener de este anlisis es que no necesitamos gran informacin de caracteres escritos: incluso con siete puntos que describan al nmero, se puede saber con error mnimo cul caracter es. Asimismo, dada la "alta" accuracy del modelo con un solo dato (ms de 40%), si se implementa un algoritmo en dos pasos: un primero que, con el primer punto, obtenga los dgitos ms probables (reducira a menos de la mitad de los dgitos posibles) y despus hiciera una prediccin basada en un modelo entrenado solamente con el subconjunto de nmeros que empiezan en ese punto, probablemente sera ms eficiente, e incluso ms rpido. Tal vez as se podra implementar ms fcilmente la red neuronal, que tom demasiado tiempo con los datos originales.
